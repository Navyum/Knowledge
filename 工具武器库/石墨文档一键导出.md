# 石墨文档怎么批量导出？被逼无奈，我自己手搓了一个 Chrome 插件（附下载）

## 写在前面

这篇文章不是什么官方通稿，纯粹是作为一个石墨文档的老用户，来聊聊最近遇到的一个大坑，以及我是怎么填上的。

如果你也像我一样，在石墨文档里存了几百上千个文件，想要把它们下载到本地备份，或者迁移到其他软件（比如 Notion、Obsidian），那你可能会遇到跟我一样的崩溃时刻。

## 我的“灾难”现场

事情是这样的，我用石墨文档已经好几年了。从工作会议记录，到个人的读书笔记、灵感清单，零零总总加起来，怎么也有几百个文档了。石墨确实挺好用的，在线协同很方便，这也是我一直用的原因。

前段时间，我心血来潮想把这些资料做个本地备份（毕竟数据拿在自己手里才踏实嘛），或者说是单纯想整理一下文件结构。

<p align="center"><img src="https://raw.staticdn.net/Navyum/imgbed/pic/IMG/6e37688a0776bb1b8cf645529781e073.png" width="80%"></p>


当我兴致勃勃地选中文件夹，准备找“批量导出”或者“打包下载”的按钮时，我傻眼了。

**没有。真的没有。**

石墨文档居然不支持批量导出！如果你想把文件下载下来，唯一的办法就是：打开一个文档 -> 点击右上角 -> 导出 -> 选择格式 -> 确认。

想象一下，几百个文件，我要重复这个动作几百次。这已经不是由于“懒”而不做了，这简直就是“赛博苦力”。

## 寻找解决办法

作为一个不愿意做重复劳动的“懒人”，我的第一反应肯定是上网搜。

搜了一圈，发现有这个痛点的人真不少。网上也有一些大神给出的方案，大多是 Python 脚本或者一些比较极客的工具。

我研究了一下，虽然我自己能看懂代码，但说实话，门槛真的挺高的：
1. 需要配置运行环境。
2. 需要自己抓包获取 Cookie 或 Token。
3. 很多脚本已经很久没维护了，接口可能都变了。

对于只是想简单备个份的朋友来说，这简直是劝退。我就想简简单单点个按钮把东西拿回来，怎么就这么难？

## 求人不如求己

既然市面上没有好用的、傻瓜式的工具，那就自己做一个吧。

于是，我花了一些业余时间，开发了一个 Chrome 浏览器插件。我的初衷很简单：**让不懂技术的人，也能一键把自己的心血批量拿回来。**

这个插件就叫 **Shimo Export**（暂定名，大家搜插件ID或者下文链接即可）。

## 它能干什么？

功能非常纯粹，没有花里胡哨的东西：

1.  **一键批量导出**：不用一个个点，支持多选或者全选。
2.  **保留原有结构**：导出的时候尽量不乱码，保持格式。
3.  **傻瓜式操作**：安装好插件，登录你的石墨文档网页版，就能看到导出按钮了。

（此处建议插入一张插件运行时的截图，例如 `asserts/main.png` 或 `asserts/download.png`，展示勾选文件和下载按钮）

## 怎么使用？

1.  下载并安装插件（链接在文末）。
2.  打开石墨文档网页版（shimo.im）。
3.  在你的文件列表页面，你会发现多了一个插件的操作面板。
4.  勾选你想导出的文件，点击“批量导出”。
5.  坐等下载完成。

## 写在最后

做这个小工具完全是出于“自救”，顺便觉得可能帮到大家。

如果你也正被石墨文档无法批量导出折磨，欢迎试用一下。如果有遇到什么 BUG，或者有什么改进建议，欢迎在评论区告诉我，或者去 GitHub 上给我提 Issue。

毕竟，我们的数据，应该由我们自己自由支配，不是吗？
